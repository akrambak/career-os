# Tool directory submission

You're writing the listing copy for a tool / project directory (Product
Hunt, BetaList, Awesome-X lists, Indie Hackers projects, etc.). The
project being listed is **Career-OS** (or the specific user-named
project).

## Output shape

Three sections, each labelled:

```
TITLE: <max 60 chars, sentence case>
TAGLINE: <max 90 chars, one line, no period>
DESCRIPTION: <120-220 words, plain text, 2 short paragraphs>
```

That literal format makes copy-paste into the directory form easy.

## Content rules

- TITLE: project name + 3-word descriptor. ("Career-OS · AI agent for
  the job hunt.") Strip emojis.
- TAGLINE: a concrete benefit, not a feature list. ("Auto-scores remote
  job postings, drafts your outreach, tracks the pipeline." beats
  "AI-powered career tools.")
- DESCRIPTION: first paragraph = problem + who it's for (1-2
  sentences) + the agent's core loop (1 sentence). Second paragraph =
  what makes it different (open-source, real-name builder, etc.) +
  link to repo (github.com/akrambak/career-os).

## Hard rules

- Never invent users, ratings, or testimonials.
- Never use "the best", "revolutionary", "game-changing".
- No bullet lists. Directories render them inconsistently.

## Return

The three labelled sections, nothing else.
